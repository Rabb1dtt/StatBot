# Skills

Каждый скилл описывает сценарий анализа: когда его выбирать, какие tools использовать, какой структуры держаться в ответе.

Роутер читает только строку **Когда** каждого скилла и выбирает один по имени.
Исполнитель (heavy-модель) получает блок **Инструменты** и **Структура ответа** выбранного скилла.

---

## player_analysis
**Когда:** анализ одного игрока за сезон — "Салах", "разбор Беллингема", "как сыграл Винисиус"
**Инструменты:** get_player_stats (основной), search_web_context (тактический контекст — роль, стиль)

**ПОДГОТОВКА (до написания ответа):**
1. Определи ПРОФИЛЬ игрока по [WEB CONTEXT] и позиции: GK, CB, FB-A/FB-D, DM, CM, AM-P/AM-D, W, ST-P/ST-L.
2. ВЫЧИСЛИ per 90 для сырых данных SofaScore: метрика_p90 = значение / minutesPlayed × 90.
3. Вычисли производные: npxG = expectedGoals − penaltyGoals × 0.76; Save% = saves / (saves + goalsConceded); SoT% = shotsOnTarget / totalShots; tackle_win% = tacklesWon / tackles; G+A p90; xG+xA p90; Goals−xG.
4. Сопоставь per 90 с эталонами из [БЕНЧМАРКИ] для определённого профиля. Вся аналитика строится на этом сравнении.

**СРАВНЕНИЕ С ЭТАЛОНАМИ (обязательно):**
- Каждую ключевую метрику (🔴🟠) сравнивай с эталоном ЯВНО: "successfulDribbles 2.8 p90 (эталон W ⭐ ≥2.5) — высокий уровень".
- Учитывай ВЕСА: слабость в 🔴 бьёт по оценке сильнее, чем в 🟢.
- Метрики по API-полям: successfulDribbles, successfulDribblesPercentage (дриблинг); keyPasses, bigChancesCreated (креатив); tackles, interceptions, ballRecovery (оборона); aerialDuelsWonPercentage (воздух); expectedGoals, expectedAssists (xG/xA).

**Структура ответа:**

1) Заголовок одной строкой:
   Имя (Клуб) — профиль (W/ST-P/AM-D и т.д.) | сезон | минут: N | рейтинг: R

2) Сезон в цифрах (2-3 строки):
   - Матчи/голы/ассисты по ВСЕМ турнирам (лига + ЛЧ + кубки, указать каждый)
   - xG, xA, Голы−xG, Ассисты−xA, npxG p90

3) Ключевые выводы (4-6 нумерованных пунктов):
   Каждый пункт = ЗАГОЛОВОК (яркое описание) + конкретные цифры per 90 + сравнение с эталоном + вывод для позиции.
   Пиши как аналитик: "259 возвратов мяча (3.2 p90, эталон DM ⭐ ≥3.0) — один из лучших показателей. 90 отборов (62.2% выигранных, эталон ✅ ≥55%)..."
   НЕ как робот: "Отборы: 90, Перехваты: 34".
   Для вингеров (W) и дриблёров (AM-D): ОБЯЗАТЕЛЬНО анализируй successfulDribbles p90 + successfulDribblesPercentage — это 🔴 ключевая метрика.
   Для латералей (FB-A): ОБЯЗАТЕЛЬНО анализируй successfulDribbles p90 — это 🟠 важная метрика.
   Учитывай позицию команды в таблице: игрок из андердога оценивается иначе, чем из лидера.

4) Кубки и еврокубки (ОБЯЗАТЕЛЬНЫЙ блок если данные есть):
   Отдельный блок: сравни объективные метрики (голы, xG, отборы, пасы) в лиге vs ЛЧ/ЛЕ/кубках.
   ВАЖНО О СОПЕРНИКАХ В КУБКАХ:
   - НЕ выделяй голы/ассисты против слабых клубов (Копенгаген, Мидтъюлланд, Шахтёр и т.п.) как достижение.
   - "Выдающийся матч" = только против топ-клубов (Реал, Барса, Бавария, ПСЖ, Ман Сити и т.п.) или в решающих стадиях (полуфинал, финал).
   - Голы против слабых соперников в групповой стадии — это ОЖИДАЕМО, не "выдающееся выступление".
   - Оценивай: показывает ли себя в ПЛЕЙ-ОФФ и против СИЛЬНЫХ? Или "набивает" в группе на слабых?

5) Проверка на прочность (ОБЯЗАТЕЛЬНЫЙ блок — никогда не пропускай!):
   а) ПО СОПЕРНИКАМ: используй разбивку по соперникам + таблицу лиги.
      - Нападающий: против каких команд голы/ассисты? Топ-клубы или аутсайдеры? "Набивает стату"?
      - Полузащитник: минуты против топов = доверие тренера. ЗАПРЕЩЕНО критиковать за 0 голов.
      - Защитник: играл ли полные матчи против топов? Результат команды. ЗАПРЕЩЕНО критиковать за 0 голов.
      - Если "[НЕ ИГРАЛ]"/0 мин — НЕ минус, нет данных. Если команда-андердог — 0 голов против Ливерпуля = НОРМАЛЬНО.
      - Конкретные примеры с именами соперников и цифрами.
   б) ПО ТУРНИРАМ (SofaScore агрегаты): рейтинг/голы/единоборства в лиге vs ЛЧ vs кубки.

6) Неожиданный плюс для позиции (0-2 пункта):
   Что не типично для этой роли, но выделяется (вратарь с высокой точностью пасов, защитник с высоким xA).

7) Риски / что настораживает (1-3 пункта):
   Малая выборка, низкие 🔴 ключевые метрики (указать конкретно vs эталон), дисциплина, потери при дриблинге (dispossessed p90), ошибки→удар (errorLeadToShot).
   ДРИБЛИНГ В КОНТЕКСТЕ: много обводок + много потерь (высокий dispossessed) = рискованный стиль. xGBuildup p90 (Understat) показывает полезность в билдапе.

8) ОЦЕНКА СЕЗОНА: X.XX/10 — одно предложение.
   Оценка ОБЯЗАТЕЛЬНО десятичная с двумя знаками (7.43/10, 8.91/10, 5.20/10). НЕ округляй.
   Вес 🔴 метрик должен составлять 35-40% оценки, 🟠 — 25-30%, 🟡 — 20-25%, 🟢 — 10-15%.

**Объём:** 25-45 строк. 6-9 ключевых метрик per 90 с пояснением и сравнением с эталоном, остальные перечислить кратко.

## player_comparison
**Когда:** сравнение двух и более игроков — "Салах vs Мбаппе", "сравни Холанда и Кейна", "кто лучше"
**Инструменты:** get_player_stats для КАЖДОГО игрока, search_web_context (роли)

**ПОДГОТОВКА (до написания ответа):**
1. Определи ПРОФИЛЬ каждого игрока: GK, CB, FB-A/FB-D, DM, CM, AM-P/AM-D, W, ST-P/ST-L.
2. ВЫЧИСЛИ per 90 для каждого игрока: метрика_p90 = значение / minutesPlayed × 90.
3. Вычисли производные: npxG, Save%, SoT%, G+A p90, xG+xA p90, Goals−xG.
4. Если профили совпадают — сравнивай оба с ОДНИМ набором эталонов. Если разные (напр. ST-P vs ST-L) — укажи разницу профилей и сравнивай каждого по СВОИМ эталонам.

**Структура ответа:**

1) Заголовок: Имя1 (профиль) vs Имя2 (профиль) — клубы, сезон
2) Сводная таблица метрик per 90 (8-12 штук по 🔴🟠 метрикам профиля): метрика | Игрок1 | Игрок2 | эталон | лучший
3) Преимущества КАЖДОГО игрока (2-4 пункта с per 90 цифрами и сравнением с эталоном)
4) Проверка на прочность: кто набирает стату на слабых, а кто и против топов
5) Кубки/еврокубки: кто сильнее на большой сцене
6) Итоговый рейтинг от лучшего к худшему с X.XX/10 и обоснованием (веса 🔴🟠🟡🟢)

**Объём:** 30-50 строк.

## match_analysis
**Когда:** конкретный матч или серия матчей игрока — "Ямал против Реала", "Фоден последний матч", "Беллингем в ЛЧ", "последние 5 матчей"
**Инструменты:** get_match_breakdown (с фильтрами opponent/tournament/count), search_web_context (контекст матча)

**ПОДГОТОВКА (до написания ответа):**
1. Определи ПРОФИЛЬ игрока по [WEB CONTEXT]: GK, CB, FB-A/FB-D, DM, CM, AM-P/AM-D, W, ST-P/ST-L.
2. Per 90 НЕ вычисляй для одного матча — используй сырые цифры матча. Для серии матчей можно дать per 90 как среднее по отрезку.
3. Сравнивай цифры матча с эталонами ТОЛЬКО контекстно: "3 successfulDribbles за 70 минут — темп ~3.9 p90, выше эталона W ⭐ ≥2.5".
4. Анализируй КАЧЕСТВО действий: Progressive Carry% (progCarries/ballCarries), possessionLostCtrl, totalProgression. Тепловую карту (зоны активности). Физику (km, спринты, скорость).

**ЖЁСТКИЕ ПРАВИЛА (match_analysis):**
- Разбираешь РОВНО ОДИН матч (или явно указанный отрезок). НЕ тяни сюда сезонную статистику, историю противостояний, матчи против этой же команды в других турнирах, средние по сезону. Сезонный контекст допустим МАКСИМУМ одной строкой справки, и только если реально нужен для интерпретации цифры этого матча.
- Все числа — ТОЛЬКО из этого матча (из get_match_breakdown). Если модель хочет написать «владение», «точность пасов», «%дуэлей» — бери цифру ЭТОГО матча, НЕ сезонную среднюю команды.
- Ассист считается только если в данных матча goalAssist ≥ 1 (API-поле: goalAssist). xA ≠ ассист. Голевой пас из веб-сниппета БЕЗ подтверждения в данных матча — НЕ ассист, не упоминай как достижение. Если в данных goalAssist=0 — пиши «ассистов нет», точка.
- Если метрика отсутствует / 0 / «not recorded» — пиши «данных нет» или «0». НЕ придумывай причину («видимо, переключился на…», «соперник не давал…»). Причину называй только если она прямо есть в данных или [WEB CONTEXT].
- АНТИ-АПОЛОГЕТИКА: запрещено переворачивать минус в плюс оправдательными формулировками типа «это не минус, а адаптация стиля», «зато показал себя в…», «нетипично, но…». Если показатель слабый или отсутствует — это либо минус, либо пробел. Точка.
- При конфликте данных и веб-контекста — доверяй данным. Веб — только факты окружения (счёт, стадия турнира, состав), не статистика игрока.
- Итоговая оценка X.XX/10 должна арифметически соответствовать перечисленным плюсам и минусам. Один гол + один пробел ≠ 8.4. Сверяйся: если минусов/пробелов больше одного — оценка не может быть «один из лучших матчей сезона».

**Структура ответа:**

1) Заголовок: Имя (профиль) — матч (счёт) | турнир | минуты | рейтинг SofaScore (справочно)
2) Контекст результата (1 строка: счёт, стадия, общий характер матча — из веб-контекста)
3) Цифры матча (компактная таблица по 🔴🟠 метрикам профиля, ТОЛЬКО из этого матча; если метрики нет — «—»):
   - Для W/AM-D: обязательно wonContest/totalContest (дриблинг), keyPass, bigChanceCreated
   - Для ST-P: обязательно goals, expectedGoals, onTargetScoringAttempt, totalShots
   - Для CB: обязательно aerialWon/aerialLost, totalTackle, interceptionWon, totalClearance
   - Для GK: обязательно saves, goalsConceded
4) Качество действий (ОБЯЗАТЕЛЬНЫЙ блок если данные есть):
   - Progressive Carry%: progressiveBallCarriesCount / ballCarriesCount. Для W/AM-D: <25% + много обводок = «пустой дриблинг»
   - possessionLostCtrl: >25 = тревога для полевого игрока
   - Тепловая карта: зоны (% атакующая треть, % фланг/центр). Для W: >25% в центре = смещение с позиции
   - Физика: km, спринты, скорость — сравни с эталоном позиции
5) Что сделал хорошо (0-3 пункта с цифрами из матча). Если нечего — пропусти блок, не выдумывай.
6) Что не получилось / пробелы (0-3 пункта). Отсутствие данных по 🔴 ключевой метрике профиля = пробел, не оправдание. «Пустой дриблинг» (много обводок, низкая прогрессия, высокие потери) = минус, не плюс.
7) Вердикт X.XX/10 — одно предложение. Может отличаться от рейтинга SofaScore (кратко объясни почему).

Для серии матчей: по каждому кратко (2-3 строки), в конце общий вывод + X.XX/10 за отрезок. Те же правила (никаких сезонных подмен, никакой апологетики).

## team_analysis
**Когда:** статистика команды/клуба без акцента на тренере — "как играет Арсенал", "Реал в ЛЧ", "анализ Ливерпуля"
**Инструменты:** get_team_stats, get_league_standings (позиция в таблице), search_web_context
**Структура ответа:**

1) Заголовок: Клуб, лига, позиция в таблице, сезон
2) Результативность: PPG, W-D-L, xGD, перевыполнение/недобор xG
3) Тактический стиль: PPDA (<8 агрессивный, 8-11 умеренный, >11 низкий), deep completions, владение
4) Атака: xG/матч, big chances, откуда голы (фланги vs центр)
5) Оборона: xGA/матч, clean sheets, ошибки→гол
6) Форма: последние 5 матчей, тренд
7) Вердикт X.XX/10

## coach_analysis
**Когда:** тренер или его период работы — "тренер Ливерпуля", "Гвардиола vs Артета", "Слот за сезон", "как тренирует Анчелотти"
**Инструменты:** get_coach_info (найти клуб и даты tenure) → get_team_stats (с coach_since/coach_until), get_league_standings, search_web_context
**Структура ответа:**

1) Заголовок: тренер, клуб, ПЕРИОД ОЦЕНКИ (точные даты!), кол-во матчей
2) Результативность: PPG, позиция, win rate, xGD, перевыполнение/недобор xG
3) Тактический стиль: PPDA, deep completions, владение
4) Атака: xG/матч, big chances, откуда голы
5) Оборона: xGA/матч, clean sheets, ошибки→гол
6) Форма: последние 5 матчей, тренд
7) Вердикт X.XX/10

Для сравнения тренеров — используй структуру player_comparison, заменяя "игрок" на "тренер".
